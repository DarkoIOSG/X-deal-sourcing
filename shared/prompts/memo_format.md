# Memo Format — IOSG 7-Section Briefing

Read shared/prompts/SKILL_en.md for full writing style, IOSG context, and quality rules.
Gold-standard examples: shared/prompts/example-jigsaw_en.md and shared/prompts/example-fireworks-7section_en.md
Blocking template: shared/prompts/output-template_en.md

## Structure

Always English. Always exactly these 7 bullets. No standalone conclusion section.

```
## [{Project name}]({official site}) 🟢/🟡/🔴

- **One-liner intro:** what it does + who it sells to. Name product lines if >1.
- **Value proposition:** pain point + how solved + one analogy ("selling shovels to X"). Must include concrete numbers.
- **Target customer:** short-term / mid-term / long-term profiles; name rep customers/types.
- **Alternatives:** 3–6 comps. Each **bolded** + one-sentence positioning + latest funding/valuation. Cover direct, indirect, DIY.
- **Sector capitalization status:** total $ raised in sector; 3–6 named comps + latest funding + investors. Data only, no opinions.
- **VC thesis:** why VCs bet on this sector + why now (2025–26) + historical analogy. Sector-level — not about this company.
- **Differentiation:** this co. vs. field — tech/team/channel/model/geo. End with Risks: (1)(2)(3) + Upsides: (a)(b)(c). Risks must be falsifiable. Final sentence: PASS / WATCH / TAKE_MEETING + one-line rationale.

Sources:
- [{name}](URL) — one line on what it contributed
```

## Confidence label
- 🟢 site + funding + team all publicly verifiable
- 🟡 mainly website + news, gaps remain
- 🔴 mostly speculation — in-person meeting mandatory
